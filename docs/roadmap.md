# 开发路线图

## 视觉风格定义

参考图的核心视觉语言可以概括为：

- 暗色底盘：整体采用接近黑曜石的深灰黑背景
- 悬浮玻璃卡片：高圆角、半透明、弱描边、柔和投影
- 青绿霓虹主色：承担主要状态、进度与主按钮
- 柔紫高光辅色：用于重点格式、选中态与层次对比
- 数据面板式排版：大量卡片化信息块，但保持留白和秩序

桌面版不会照搬手机图的拥挤布局，而是保留它的气质，重做成更适合桌面下载器的版本：

- 核心操作放在左侧主区
- 作品预览和格式选择放在右侧
- 任务列表保持高信息密度但低视觉噪音
- 动效只服务状态反馈，不做花哨噪声

## 模块拆分

### 前端

- `Shell`：窗口框架、导航、全局布局
- `Analyzer`：分享文本输入、解析状态、错误提示
- `FormatPicker`：格式列表、无水印标签、登录要求
- `TaskQueue`：当前任务、历史任务、进度与状态
- `Preferences`：保存目录、命名模板、并发配置
- `Account`：登录态提示、会话恢复、失效提醒

### 后端

- `parser`：提取分享文案中的真实 URL
- `resolver`：短链跳转、作品 ID 提取
- `extractor`：作品信息、格式列表、无水印能力识别
- `auth`：Cookie/登录态管理、安全存储
- `downloader`：下载、暂停、恢复、重试、校验
- `queue`：任务排队、并发控制、任务事件分发
- `storage`：SQLite 持久化、下载历史、设置

## 迭代顺序

### 阶段 1：界面与骨架

- 完成 Tauri + Svelte 工程搭建
- 完成桌面首页与格式选择界面
- 定义前后端数据模型与命令接口

### 阶段 2：单条下载 MVP

- 输入分享文本并解析短链
- 抓取公开作品元数据
- 展示可用格式
- 选择格式并完成单条下载

### 阶段 3：登录态与高级清晰度

- 独立登录窗口
- 安全保存 Cookie / 会话信息
- 登录后刷新格式列表
- 区分公开格式与高级格式

### 阶段 4：稳定性

- 失败重试
- 断点续传
- 文件命名模板
- 错误分类与诊断日志

### 阶段 5：批量下载

- 多链接导入
- 剪贴板监听
- 队列优先级
- 批量暂停/恢复
