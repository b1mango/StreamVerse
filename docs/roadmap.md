## 当前产品形态

### 单视频下载

- 输入分享文案 / 短链 / 作品链接
- 解析作品与清晰度
- 勾选下载内容
- 选择格式并开始下载
- 任务区查看进度、速度、ETA 与状态控制

### 主页批量下载

- 输入主页链接
- 打开浏览器窗口读取作品列表
- 勾选需要下载的作品
- 统一入队下载
- 在任务区统一暂停、继续或取消

## 视觉方向

产品当前采用“暗色极简桌面工具”方向，而不是早期更偏装饰化的仪表盘风格。

保留这些视觉特征：

- 深色底盘与克制的玻璃感
- 青绿色主强调色
- 温和圆角与低噪声分层
- 清晰的信息层级与充足留白

持续优化的目标：

- 进一步减少无效装饰
- 强化页面切换后的任务心智模型
- 持续补面向 GitHub 展示的截图与说明图

## 架构方向

### 前端

- 当前已经拆出平台首页、单视频页、批量页、任务队列、设置面板等独立组件
- 下一步继续收敛跨平台共享逻辑，降低 [src/App.svelte](../src/App.svelte) 作为总控入口的复杂度

### 后端

- `media_contract`：主应用与 pack 共享的数据契约
- `providers`：统一平台入口与工作流分发
- `pack_host`：平台 pack 调度与回包解析
- `pack_manager`：模块安装、卸载、更新、共享依赖安装
- `pack_registry`：模块包与共享依赖包映射关系
- `parser`：分享文案 URL 提取
- `formats`：格式去重与偏好选择
- `ytdlp`：共享下载执行、任务控制与 sidecar 产物保存
- `settings`：下载目录、登录源与偏好持久化

## 当前主要风险

- 抖音网页接口与风控可能波动
- 不同链接类型对 Cookie 的依赖程度不一致
- Bilibili 高质量 DASH 依赖 FFmpeg
- 浏览器窗口批量读取依赖本机可用的 Chromium 内核浏览器
- Windows 虽有代码路径，但还缺实机验证
